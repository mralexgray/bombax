Detailed Description

Convenience handler for serving static files.

Author:
Bombaxtic LLC - http://www.bombaxtic.com
Since:
1.0

Although it is much more efficient to use static folders configured in the Bombax server, this handler returns static files to the client with support for a large variety of MIME types and automatic removal of location prefixes. By default BxStaticFileHandler is mapped to the special static resource folder as described in BxApp's staticWebPath documentation.

Note:
The most common use of BxStaticFileHandler is to transparently support accessing the static resources folder during debugging.

Example of enabling the BxStaticFileHandler in a BxApp: 

 - (id)setup {
     [BxStaticFileHandler setStaticResourcePath:@"/path/to/uploads"];
     [self setHandler:@"BxStaticFileHandler" forPrefix:@"uploads"];
     return self;
 }

+ (void) setStaticResourcePath: (NSString *)  path  

sets the root path for BxStaticFileHandler requests

By default the your BxApp's static resource folder is used.

Warning:
Because the handler is instantiated and used based on its class name, it is not normally possible to have multiple BxStaticFileHandler roots.

Example of changing the static resource path: 

 - (id)setup {
     [BxStaticFileHandler setStaticResourcePath:@"/path/to/uploads"];
     [self setHandler:@"BxStaticFileHandler" forPrefix:@"uploads"];
     return self;
 }
